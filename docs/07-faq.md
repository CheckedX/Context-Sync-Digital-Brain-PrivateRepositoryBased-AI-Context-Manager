# 常见问题

## Token 相关

### Q: Token 泄露了怎么办？

立刻删除并重新生成：

1. GitHub → Settings → Developer settings
2. Personal access tokens → Tokens (classic)
3. 找到泄露的 Token → Delete
4. Generate new token

删除后，旧的 Token 立刻失效。

### Q: 多久换一次 Token？

建议 90 天换一次，或者当你觉得可能泄露时立刻换。

### Q: Token 存在哪最安全？

- 让你的 AI 存（它会加密保存）
- 或者用密码管理器（1Password、Bitwarden）
- 不要存在公开文档、聊天记录里

## 仓库相关

### Q: 私有仓库安全吗？

比你想象的更安全：
- 只有你能看到
- GitHub 有完善的安全机制
- 你的 Token 权限只限于你授权的仓库

### Q: 仓库名能改吗？

可以。改完后告诉 AI 新的仓库名就行。

### Q: 误删了文件怎么办？

GitHub 有历史记录，可以恢复：
1. 进入仓库
2. 点击某个文件的历史
3. 找到删除前的版本
4. 恢复

## 使用相关

### Q: 每次都要说"commit"吗？

不一定要。你可以设置：
- AI 自己判断后直接 commit
- AI 整理完后给你确认
- 定时提醒你整理

### Q: 我能手动上传吗？

可以。随时登录 GitHub 手动编辑或上传文件。AI 自动化只是更省事。

### Q: 换了一个 AI 怎么办？

你的 context 在 GitHub，不属于任何一个 AI。

把仓库地址和 Token 给新的 AI，它就能继续用。

### Q: AI 怎么知道记什么？

AI 有自己的判断标准（方法论、决策、项目经验等）。

你也可以：
- 明确告诉 AI"这个记一下"
- 或者让 AI 列出候选，你选择

## 其他问题

### Q: 这和 MemPalace 有什么区别？

| 特点 | GitHub Context | MemPalace |
|------|----------------|-----------|
| 方式 | 手动/半自动 | 全自动 |
| 存储 | GitHub 云端 | 本地 |
| 搜索 | 关键词 | 语义搜索 |
| 门槛 | 低 | 中高 |
| 适合 | 刚开始的人 | 重度用户 |

### Q: 会不会信息过载？

不会，如果你遵循"简洁 > 完备"原则。

少记一点，记住重要的。

### Q: 积分/费用问题？

GitHub 私有仓库免费。

AI 对话可能消耗积分，但整理和 commit 本身不额外消耗。
