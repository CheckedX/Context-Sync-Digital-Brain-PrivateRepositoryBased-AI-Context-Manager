# 进阶玩法

## 1. 使用模板仓库 Fork

### 什么是模板仓库？

一个已经建好结构的仓库，你 Fork 后直接用，不用从零开始。

### 怎么用？

1. 找到模板仓库（比如本项目）
2. 点击 `Fork` 按钮
3. 你的账号下就有了一个一模一样的仓库
4. 把示例内容替换成你自己的
5. 配置 Token，开始使用

---

## 2. 创建多个仓库

### 场景

- `my-context` - 个人 context
- `work-context` - 工作 context
- `project-xxx` - 特定项目 context

### 怎么做？

每个仓库单独配置，告诉 AI 要往哪个仓库 push。

---

## 3. 与其他工具集成

### MemPalace

如果你是重度用户，可以安装 MemPalace：
- 本地存储所有对话
- 语义搜索
- 更强大的记忆能力

**适合**：每天大量对话、需要精细搜索的人

### Coze 对话导出

如果你的 AI 在 Coze：
- 网页端可以导出为 Markdown/PDF
- API 可以导出为 JSON
- 让 AI 帮你转换格式

---

## 4. 自定义记录模板

### 为什么需要模板？

统一格式，方便搜索和理解。

### 示例模板

**项目记录模板**：
```markdown
# 项目名称

## 目标

## 我的角色

## 技术栈

## 关键收获

## 后续计划
```

**决策记录模板**：
```markdown
# 决策名称

## 决策内容

## 决策理由

## 考虑过的其他选项

## 后续验证
```

### 怎么让 AI 用模板？

告诉 AI：

> "以后记录项目，用这个模板..."

然后把模板发给它。

---

## 5. 跨 AI 使用

### 场景

你用多个 AI（Claude、GPT、Gemini...），想让它们共享 context。

### 怎么做？

1. Context 存在 GitHub
2. 把仓库地址和 Token 给每个 AI
3. 它们都能读写你的 context

### 注意

不同 AI 的记忆方式可能不同，需要适配。

---

## 6. 自动化脚本

### 场景

你想让 AI 自动 commit，不用每次确认。

### 怎么做？

告诉 AI：

> "以后你觉得值得记的内容，直接 commit，不用问我。"

AI 会自动判断和执行。

### 风险

可能会记录一些你不想要的内容。

建议先用确认模式，熟悉后再开全自动。

---

## 总结

进阶玩法的核心是**找到适合自己的节奏**：

- 刚开始 → 手动确认，慢慢积累
- 熟悉后 → 半自动，定时提醒
- 重度用户 → 全自动，集成其他工具

一格一格来，不要急。
