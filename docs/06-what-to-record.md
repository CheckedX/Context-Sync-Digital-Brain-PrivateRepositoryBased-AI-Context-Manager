# 什么值得记录？

## 值得记录 ✅

### 1. 方法论

你总结出来的做事方法。

**例子**：
- 如何用 GitHub 管理 context
- 如何做项目复盘
- 如何学习新技能

### 2. 决策及理由

你做过的重要决定，以及为什么这样决定。

**例子**：
- 选择 XX 职业方向，因为...
- 用 XX 工具而不用 YY，因为...

**为什么重要**：
- 未来遇到类似情况，可以回顾当时的思考
- AI 知道你的决策逻辑，能更好地帮你

### 3. 可复用知识

可以多次使用的知识、技巧、术语。

**例子**：
- 行业术语（Token, API, Context...）
- 工具使用技巧
- 代码片段

### 4. 项目经验

你做过或正在做的项目。

**记录内容**：
- 项目目标
- 你的角色
- 技术栈
- 学到的经验
- 后续优化方向

### 5. 关键洞察

突然想到的点子、顿悟、灵感。

**例子**：
- "Context 是你和 AI 共同知道的东西"
- "完美主义阻止我展示成果"

## 不值得记录 ❌

| 类型 | 例子 |
|------|------|
| 日常闲聊 | "今天天气不错" |
| 临时操作 | "帮我查个快递" |
| 无沉淀价值 | 纯娱乐内容 |

## 我的判断标准

问自己一个问题：

**"这段内容，未来还有用吗？"**

- 有用 → 记
- 没有 → 不记

不确定的，先记下来。以后可以删。

## 记录原则

**简洁 > 完备**

宁可漏一点，不要记太多。

太多信息会稀释真正重要的内容。

一格一格来，慢慢积累。
